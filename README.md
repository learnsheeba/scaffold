# SwiftChat

A local-network, end-to-end encrypted chat system built entirely in Swift & SwiftUI.
See [`SYSTEMDESIGN.md`](./SYSTEMDESIGN.md) for the full architecture.

## Targets

| Target | Type | Description |
|---|---|---|
| `SwiftChatKit` | library | Shared cross-platform logic: CryptoKit E2EE, wire protocol, SwiftData models, chess engine, FoundationModels summarizer, FileManager media store. |
| `SwiftChatUI` | library | Shared SwiftUI: Path-drawn bubbles, 5 animated backgrounds, SpriteKit chess, typing indicator, link previews. |
| `SwiftChatServer` | executable | Vapor WebSocket **relay**. Routes opaque ciphertext only — never links crypto, never decrypts. |
| `SwiftChatIOS` / `SwiftChatMac` | apps | The two client apps (`@main` App structs). |

## Requirements

- macOS 15+ / iOS 18+ (needed for SwiftData, FoundationModels, `PhaseAnimator`).
- Xcode 16+ with the Swift 6 toolchain.
- Apple Intelligence-capable device for on-device summaries (graceful fallback otherwise).

## Running

### 1. Start the relay server (on a Mac on the LAN)

```bash
swift run SwiftChatServer
# listens on ws://0.0.0.0:8080/chat
```

### 2. Run the tests

```bash
swift test            # runs SwiftChatKitTests (crypto, models, wire, chess)
```

> The XCUITest (`Tests/SwiftChatUITests`) requires an app host and runs from the
> Xcode project's UI-testing bundle, not from `swift test`.

### 3. Build the client apps

The iOS/macOS apps ship as SwiftPM library sources plus `@main` entry points,
wired into an Xcode project generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from [`project.yml`](./project.yml):

```bash
brew install xcodegen
xcodegen generate                 # creates SwiftChat.xcodeproj
open SwiftChat.xcodeproj          # then build/run SwiftChatIOS or SwiftChatMac
```

`SwiftChat.xcodeproj` is git-ignored (generated); regenerate it any time with
`xcodegen generate`.

## Continuous Integration (build on a macOS runner — no local Mac needed)

Since the apps require Apple frameworks, CI runs on GitHub's **macOS runners**.
The workflow at [`.github/workflows/ci.yml`](./.github/workflows/ci.yml) has three jobs:

1. **swiftpm** — `swift build` + `swift test --filter SwiftChatKitTests` (crypto, models, wire, chess, SwiftData).
2. **xcode-apps** — `xcodegen generate`, then `xcodebuild` the **iOS** (simulator) and **macOS** apps.
3. **ui-tests** — runs the XCUITest on an iPhone 16 simulator.

To trigger it from your Windows machine:

```bash
git init
git add .
git commit -m "SwiftChat: initial implementation + CI"
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main
```

Then open the repo's **Actions** tab to watch the macOS VM build and test.
You can also run it on demand via the **workflow_dispatch** trigger.

> Note: macOS runner minutes count 10× against the free quota on private repos.
> Public repos get free macOS minutes.

Both clients connect to the relay, perform an X25519 handshake, and exchange
AES-GCM (or ChaChaPoly) sealed payloads. The server sees only ciphertext.
